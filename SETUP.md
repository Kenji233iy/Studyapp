# StudyFlow - Complete Setup Guide

## Quick Start (5 Minutes)

### 1. Install Dependencies
```bash
# Install all dependencies (root + frontend + backend)
npm install
cd frontend && npm install && cd ..
cd backend && npm install && cd ..
```

### 2. Create Environment Files

**frontend/.env.local**
```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

**backend/.env.local**
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
ANTHROPIC_API_KEY=sk-ant-...
```

### 3. Setup Supabase Database

1. Create account at https://supabase.com
2. Create new project
3. Go to SQL Editor → New Query
4. Copy entire contents of `/database.sql` and execute
5. Create Storage bucket named `pdfs` (Settings → Storage)
6. Get API keys from Settings → API

### 4. Run Locally

**Option A: Run Both (from root)**
```bash
npm run dev
```
Frontend: http://localhost:5173
Backend: http://localhost:3000

**Option B: Separate Terminals**
```bash
# Terminal 1 - Frontend
cd frontend && npm run dev

# Terminal 2 - Backend  
cd backend && npm run dev
```

## 🔌 Key Files Explained

### Frontend Components

**DashboardLayout.tsx** - Main layout wrapper
- Sidebar (260px fixed)
- Topbar (with settings)
- Content area (flex-1)
- Authentication check

**Sidebar.tsx** - Navigation & widgets
- Logo + app title
- Nav items (Upload, Summaries, Flashcards, etc)
- Pomodoro widget
- Task list (5 most recent)
- "IA Conectada" status indicator

**SettingsModal.tsx** - User preferences
- Theme, language, AI density
- Pomodoro durations
- Sound & auto-start toggles
- Syncs to Supabase

**Pomodoro.tsx** - Timer widget
- SVG ring progress indicator
- 4 session types (focus, short/long break, long break)
- Auto-switch between modes
- Browser notifications

### Frontend Pages

**UploadPage.tsx** - PDF file upload
- Drag-and-drop zone
- Multiple file selection
- Auto-upload to Supabase Storage
- Display uploaded documents

**SummariesPage.tsx** - Study summaries
- Generated by Claude
- Filter by document/topic
- Edit inline
- Copy to clipboard

**FlashcardsPage.tsx** - Spaced repetition
- 3D flip animation
- Study mode (full-screen)
- SM-2 algorithm for scheduling
- Filter by difficulty/status

**KeypointsPage.tsx** - Key concepts
- Core, exam, trap types
- Filter by importance
- Related concepts links
- Export to text

**ChatPage.tsx** - AI assistant
- 50/50 split: PDF viewer + chat
- Streaming responses
- Context-aware answers
- Generate material from chat

### Zustand Stores

**documentStore** - Uploaded PDFs
**summaryStore** - Generated summaries
**flashcardStore** - Flashcard data + spaced repetition
**keypointStore** - Key points with filters
**taskStore** - Task list + reordering
**settingsStore** - User preferences (persisted)

### Backend API Routes

```
POST /api/analyze-pdf
  ↓ Extracts text from PDF
  ↓ Sends to Claude API
  ↓ Returns: { topics, documentSummary }

POST /api/generate/summaries
  ↓ Claude generates summaries
  ↓ Returns: { summaries[] }

POST /api/generate/flashcards
  ↓ Claude creates study cards
  ↓ Returns: { flashcards[] }

POST /api/generate/keypoints
  ↓ Claude extracts key concepts
  ↓ Returns: { keypoints[] }

POST /api/chat
  ↓ Streaming chat with context
  ↓ Returns: EventSource stream
```

- Body: DM Sans (Google Fonts)

**Components**
- All use `.glass` class

- Border: 1px solid rgba(255,255,255,0.85)

## 🔐 Authentication Flow

1. User arrives at app → Redirected to LoginPage
2. LoginPage shows Supabase Auth UI
3. User signs up/logs in (email or Google OAuth)
4. Session stored in browser
5. Redirected to /upload (main dashboard)
6. User preferences created in DB
7. All data filtered by user_id (RLS)
8. Logout clears session

## 💾 Database Structure

### Tables

**documents**
- id, user_id, name, storage_path, topics, page_count, created_at

**summaries**
- id, user_id, document_id, topic, title, body, tags, key_terms, created_at, updated_at

**flashcards**
- id, user_id, document_id, question, answer, category, difficulty
- status ('new'|'review'|'mastered')
- next_review_at, ease_factor, interval (SM-2 algorithm)

**keypoints**
- id, user_id, document_id, topic, type ('core'|'exam'|'trap')
- text, related_concepts, source_hint

**chat_messages**
- id, user_id, session_id, role ('user'|'assistant')
- content, document_id, page_reference

**tasks**
- id, user_id, text, priority ('alta'|'media'|'baja')
- done, due_date, source ('manual'|'ai'), position

**pomodoro_sessions**
- id, user_id, duration_minutes, type ('focus'|'short_break'|'long_break')
- completed, document_id

**user_preferences**
- user_id (PK), pomodoro_focus, theme, language, ai_density, sound_enabled, etc.

## 🐛 Common Issues & Fixes

| Issue | Fix |
|-------|-----|
| "Cannot find env variables" | Ensure `.env.local` files exist and restart dev server |
| CORS errors | Make sure backend runs on 3000, frontend proxy is configured |
| Supabase connection fails | Check API keys are correct and RLS policies are enabled |
| Claude API errors | Verify ANTHROPIC_API_KEY in backend `.env.local` |
| PDF upload fails | Ensure `pdfs` bucket exists in Supabase Storage |
| Flashcards not saving | Check that `flashcards` table exists and RLS policy allows inserts |
